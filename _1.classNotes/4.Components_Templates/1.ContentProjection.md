# Content Projection

Content projection is a powerful feature in Angular that allows you to insert dynamic content into a component from outside that component. This is particularly useful for creating reusable components that can display different content based on the context in which they are used.

## Basic Content Projection

To implement basic content projection, you use the `<ng-content />` directive in your component's template. This directive acts as a placeholder for the content that will be projected into the component.

```typescript
import { Component, Input } from "@angular/core";
@Component({
  selector: "app-dashboard-item",
  templateUrl: "./dashboard-item.component.html",
  standalone: true,
})
export class DashboardItemComponent {
  @Input() title!: string;
  @Input() image!: { src: string; alt: string };
}
```

```html
<div class="dashboard-item">
  <article>
    <header>
      <img [src]="image.src" [alt]="image.alt" />
      <h2>{{ title }}</h2>
    </header>
    <ng-content />
  </article>
</div>
```

## Multiple Slots Content Projection

Sometimes, you may want to project different pieces of content into specific locations within your component. This can be achieved using named slots with the `select` attribute on the `<ng-content />` directive.

```typescript
import { Component, Input } from "@angular/core";
@Component({
  selector: "app-dashboard-item",
  templateUrl: "./dashboard-item.component.html",
  standalone: true,
})
export class DashboardItemComponent {
  @Input() title!: string;
  @Input() image!: { src: string; alt: string };
}
```

```html
<div class="dashboard-item">
  <article>
    <header>
      <img [src]="image.src" [alt]="image.alt" />
      <h2>{{ title }}</h2>
    </header>
    <section class="details">
      <ng-content select=".details" />
    </section>
    <section class="actions">
      <ng-content select=".actions" />
    </section>
  </article>
</div>
```

## ngProjectAs

In some cases, you might want to project content that doesn't match the selectors defined in your component. The `ngProjectAs` attribute allows you to specify how the content should be treated for projection purposes.

```html
<app-dashboard-item
  title="Server Status"
  [image]="{ src: 'assets/server.png', alt: 'Server Image' }"
>
  <div ngProjectAs=".details">
    <p>All systems operational.</p>
  </div>
  <div ngProjectAs=".actions">
    <button>View Details</button>
  </div>
</app-dashboard-item>
```

## View Encapsulation

By default, Angular uses view encapsulation to scope styles to a specific component. However, when using content projection, you might want to adjust the encapsulation strategy to ensure that styles are applied correctly to projected content. You can set the `encapsulation` property in the `@Component` decorator to control this behavior.

```typescriptimport { Component, Input, ViewEncapsulation } from "@angular/core";
@Component({
  selector: "app-dashboard-item",
  templateUrl: "./dashboard-item.component.html",
  standalone: true,
  encapsulation: ViewEncapsulation.None, // or ViewEncapsulation.ShadowDom
})
export class DashboardItemComponent {
  @Input() title!: string;
  @Input() image!: { src: string; alt: string };
}
```

- `ViewEncapsulation.None`: This setting disables view encapsulation, allowing styles defined in the component to affect projected content and vice versa.
- `ViewEncapsulation.ShadowDom`: This setting uses the browser's native Shadow DOM to encapsulate styles, providing strong isolation between the component and projected content.ShadowDom.
- `ViewEncapsulation.Emulated`: This is the default setting, which emulates Shadow DOM behavior by adding unique attributes to styles and elements.
